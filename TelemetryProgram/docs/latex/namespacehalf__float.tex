Cast to or from half-\/precision floating point number. This casts between \doxylink{classhalf__float_1_1half}{half} and any built-\/in arithmetic type. Floating point types are converted via an explicit cast to/from {\ttfamily float} (using the rounding mode of the built-\/in single precision implementation) and thus any possible warnings due to an otherwise implicit conversion to/from {\ttfamily float} will be suppressed. Integer types are converted directly using the given rounding mode, without any roundtrip over {\ttfamily float} that a {\ttfamily static\+\_\+cast} would otherwise do. It uses the default rounding mode.

Using this cast with neither of the two types being a \doxylink{classhalf__float_1_1half}{half} or with any of the two types not being a built-\/in arithmetic type (apart from \doxylink{classhalf__float_1_1half}{half}, of course) results in a compiler error and casting between \doxylink{classhalf__float_1_1half}{half}s is just a no-\/op.
